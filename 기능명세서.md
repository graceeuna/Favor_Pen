## 1. 문서 개요

### 1.1 목적
Epic Pen(데스크탑 화면 위에 그림/주석을 그리는 항상-위 투명 오버레이 도구)의 핵심 기능을 조사하고, 이를 **설치 과정 없이 실행되는(무설치) 자체 프로그램**으로 재현하기 위한 기능 요구사항을 정의한다.

### 1.2 제품 한 줄 정의
> "어떤 화면(바탕화면·발표 슬라이드·영상·웹·앱) 위에든 즉시 펜·도형·텍스트로 주석을 그리고, 필요하면 화면을 통과시켜 아래 앱을 그대로 조작할 수 있으며, USB에 담아 어디서나 설치 없이 실행되는 강의·발표·캡처용 화면 주석 도구."

### 1.3 Epic Pen 대비 차별화 방침
- Epic Pen은 핵심 기능(텍스트·도형·화이트보드·고스트 모드 등)을 **Pro 유료 구독(€2/월, 연 €24, 일회성 구매 없음)** 으로 분리한다. [근거 1]
- **본 제품은 라이선스/구독 분기 없이 Epic Pen 무료+Pro 기능 전부를 단일 무료 무설치 빌드로 제공**하는 것을 기본 방침으로 한다. (구독 게이팅 미구현 → 구현 단순화)

## 2. Epic Pen 조사 요약 (원본 기능 전수)

### 2.1 제품 성격
Epic Pen은 Windows 데스크탑 위에 **항상-위(always-on-top) 투명 오버레이**를 띄워, 실행 중인 다른 앱과 무관하게 화면 어디에든 그림/글씨/하이라이트를 그릴 수 있는 주석 도구다. v3.9.95(2020-11-25)부터 **백그라운드 서비스(상주)** 로 동작하여 툴바가 숨겨진 상태에서도 단축키로 도구를 쓸 수 있다. [근거 5]

### 2.3 공통/시스템 기능
- **멀티모니터**: 여러 화면에 걸쳐 실시간 그리기/쓰기/하이라이트. 화이트보드/블랙보드는 모든 화면이 아닌 **특정 단일 화면에만** 표시 지정 가능(v3.11.0, 2022-04-08). [근거 6]
- **단축키 시스템**: 툴바 숨김 상태에서도 도구·색상·고스트 모드를 단축키로 접근. v3.9.95에서 핫키 전면 재설계. [근거 5]
- **클릭관통(click-through)**: 백그라운드 앱과 상호작용하지 않고 화면에 그릴 수 있으며, 주석을 유지한 채 데스크탑 조작으로 복귀 가능. 메커니즘은 **고스트 모드**(Settings > Ghost Mode > Enable)로 툴바를 숨기고 단축키만 사용하는 방식. [근거 7]

### 2.4 공식 문서 위치
- 사용자 가이드: `epicpen.com/userguide` (무료/Pro 기능 분리 표기) [근거 3]
- 기능 소개: `epicpen.com/features` [근거 2, 4]
- 버전 히스토리(changelog): `epicpen.com/versionhistory` [근거 4, 5, 6]
- FAQ: `epicpen.com/faq` [근거 7]
- Pro/구독: `epicpen.com/pro`, `epicpen.com/subscribe` [근거 1]

### 2.5 조사로 확정하지 못한 원본 기능 (§10 참조)
- **줌/돋보기(magnifier)**: 조사 범위에 포함했으나 명시적 1차 증거 미확보 → 본 제품에서는 "선택(Optional)" 기능으로 분류.
- 스크린샷 저장 파일 형식/경로/클립보드 동작, 주석 프로젝트 저장/불러오기(persistence) 여부: 1차 증거 미확보 → 본 제품에서 자체 설계.

## 4. 상세 기능 명세

### FR-01 투명 항상-위 오버레이 캔버스 (M)
- 화면 전체를 덮는 투명·테두리 없는 항상-위 윈도우를 띄운다.
- 픽셀 단위 알파 투명을 지원해, 그린 잉크만 보이고 나머지는 완전 투명해야 한다.
- 작업표시줄에 표시되지 않으며(ShowInTaskbar=false), Alt+Tab 목록에서 가급적 숨긴다.
- 전체화면 영상/슬라이드쇼 위에서도 오버레이가 유지되어야 한다.
- (기술: §7.1 WPF 레이어드 윈도우)

### FR-02 펜(프리핸드) 도구 (M)
- 마우스/터치/스타일러스 드래그 경로를 부드러운 폴리라인(또는 베지어)으로 그린다.
- 안티에일리어싱 적용(매끄러운 잉크 엣지). [근거 9]
- 펜압 감지 지원 시 굵기 가변(Optional, 입력장치 의존).
- 그린 결과는 벡터 스트로크 객체로 관리(개별 삭제·Undo 가능해야 함).

### FR-03 하이라이터 도구 (M)
- 반투명(예: 알파 40~50%) 굵은 스트로크.
- 같은 위치 중첩 시 색이 진해지지 않도록(동일 스트로크 내 균일 알파) 처리 권장.
- 형광 계열 기본 색(노랑·연두·분홍·하늘).

### FR-04 지우개 / 부분 지우기 / 전체 지우기 (M)
- **스트로크 지우개**: 커서가 닿은 스트로크 객체를 통째로 삭제.
- **영역 지우개**(Optional): 픽셀 영역 부분 삭제.
- **전체 지우기(Clear/Trash)**: 현재 캔버스의 모든 주석을 한 번에 제거. 단축키 매핑(예: `Esc` 길게 또는 전용 키). [근거 3]

### FR-05 색상 선택 (M)
- 기본 팔레트 ≥16색 제공(Epic Pen 무료 동등). v1에서 24색으로 확장. [근거 4]
- **Quick Colors**: 툴바에 빠른 색 4~6칸. v1에서 사용자 지정 가능. [근거 4]
- 커스텀 색상 피커(RGB/HEX 입력) 제공.

### FR-06 굵기 선택 (M)
- 최소 3단계(가늘다/보통/굵다) 이상, 슬라이더 권장.
- 권장 범위: 1~40 px (펜 1~20, 하이라이터 10~40). *(정확한 원본 수치는 §10-4 미확인 → 자체 정의)*

### FR-07 Undo / Redo (M)
- 스트로크/도형/텍스트 단위 실행취소·재실행 스택.
- 단축키 `Ctrl+Z` / `Ctrl+Y`(또는 `Ctrl+Shift+Z`).

### FR-08 플로팅 툴바 (M)
- 화면 위를 드래그로 이동 가능한 작은 도구 막대.
- 구성: 도구 선택(펜/하이라이터/지우개/도형/텍스트), Quick Colors, 굵기, Undo/Redo, 전체 지우기, 캡처, 화이트보드 토글, 설정, 종료.
- 멀티모니터 시 모니터 간 이동 가능해야 함(원본 v3.12.17 버그픽스 참조). [근거 6]
- 숨김/표시 토글(고스트 모드와 연동).

### FR-09 클릭관통 / 그리기 모드 토글 (M) — ★핵심
- 두 상태를 단축키/툴바로 전환한다:
  - **그리기 모드(Draw)**: 오버레이가 마우스 입력을 캡처 → 그림이 그려짐.
  - **통과 모드(Pass-through)**: 오버레이가 입력을 무시 → 아래 앱을 그대로 클릭/스크롤. 그려둔 주석은 화면에 그대로 유지. [근거 7, 14]
- 전환 시 시각적 피드백(예: 툴바 색/트레이 아이콘 변화) 필수.
- ⚠️ **설계 제약(중요)**: "그리는 동시에 아래 앱도 클릭"은 단일 상태로 동시 달성 불가. 반드시 모드 토글로 분리한다. (상세 §8.1) [근거 11]

### FR-10 전역 단축키 (M)
- 앱이 포커스가 없어도 동작하는 글로벌 핫키 등록.
- 기본 매핑(사용자 변경 가능):
  | 동작 | 기본 키 |
  |---|---|
  | 그리기/통과 모드 토글 | `Ctrl+Alt+D` |
  | 펜 / 하이라이터 / 지우개 | `Ctrl+Alt+1/2/3` |
  | 전체 지우기 | `Ctrl+Alt+E` |
  | Undo | `Ctrl+Z` |
  | 스크린샷 | `Ctrl+Alt+S` |
  | 화이트보드 토글 | `Ctrl+Alt+W` |
  | 고스트 모드 | `Ctrl+Alt+G` |
  | 종료 | `Ctrl+Alt+Q` |
- (ppInk가 글로벌 핫키 기반 클릭관통 토글을 검증함 — 기본 `Ctrl+Alt+G`. [근거 13])

### FR-11 스크린샷 캡처·저장·클립보드 복사 (M)
- 주석이 합성된 현재 화면을 캡처.
- 저장 형식 PNG(투명 보존 옵션) / JPG, 저장 경로 사용자 지정 + 클립보드 복사.
- 캡처 시 툴바는 결과 이미지에서 제외(촬영 직전 숨김 처리). *(원본 세부는 §10-2 미확인 → 자체 설계)*

### FR-12 트레이 상주 (M)
- 시스템 트레이 아이콘으로 상주, 우클릭 메뉴(보이기/숨기기, 설정, 정보, 종료).
- 백그라운드 상주 상태에서도 전역 단축키 동작(원본 "service" 동작 대응). [근거 5]

### FR-13 무설치 설정 영속성 (M)
- 모든 설정(색상, 굵기, 핫키, 최근 저장 경로 등)을 **실행파일과 같은 폴더의 설정 파일**(`settings.json` 또는 `.ini`)에 저장.
- 레지스트리·`%AppData%` 미사용(쓰기 권한 없는 경로 대비 fallback만 허용) → USB 이동 시 설정 함께 이동.
- 설정 파일이 없으면 기본값으로 생성.

### FR-14 도형 도구: 직선 / 화살표 / 사각형 / 타원 (v1)
- 드래그로 시작점→끝점 정의, 드래그 중 미리보기.
- `Shift` 보정: 직선 각도 스냅(15°), 정사각형/정원.
- 화살표 머리 크기 비례. [근거 4, 12]

### FR-15 텍스트 입력 도구 (v1)
- 클릭 위치에 텍스트 박스 생성, 임의 색상·크기. [근거 4]
- 폰트/크기/색상 선택, 입력 후 위치 이동.
- (ppInk는 좌/우 정렬 텍스트 지원. [근거 12])

### FR-16 화이트보드 / 블랙보드 캔버스 (v1)
- 버튼 토글로 불투명 흰색/검정 배경 캔버스를 오버레이 위에 깐다(그 위에 주석). [근거 4]
- 멀티모니터 시 **특정 단일 화면에만** 표시 옵션(설정의 whiteboard 섹션). [근거 6]
- 토글 해제 시 투명 오버레이로 복귀(주석 유지 옵션).

### FR-17 듀얼/다중 모니터 지원 (M, ★필수)
- **2개 이상의 모니터가 연결된 환경에서 모든 화면에 주석을 그릴 수 있어야 한다.** 단일 모니터 전용 동작은 출시 기준 미달.
- 모니터 경계를 넘는 연속 드로잉(한 스트로크가 화면 A→B로 이어짐) 지원. [근거 6]
- **DPI 스케일 혼합 대응(필수)**: 모니터별 배율(예: 100% + 150%)이 달라도 잉크 위치·굵기가 왜곡되지 않아야 한다 → Per-Monitor V2 DPI Awareness 적용(§8.2).
- 모니터 추가/제거·해상도 변경·배치 변경 시 오버레이가 자동으로 재배치/재구성되어야 한다(`DisplaySettingsChanged` 감시).
- 플로팅 툴바(FR-08)는 모니터 간 자유 이동 가능해야 한다.
- 화이트보드/블랙보드(FR-16)는 **특정 단일 모니터에만** 표시하도록 대상 화면 선택 가능. [근거 6]
- **구현 방식(택1, M2.2에서 확정)**:
  - (A) 가상 화면(virtual desktop) 전체를 덮는 단일 오버레이 윈도우 — 구현 단순, M0/M1 채택. 단 혼합 DPI에서 좌표 보정 필요.
  - (B) 모니터당 오버레이 윈도우 1개 — 혼합 DPI·단일화면 화이트보드에 유리(권장 최종형).

### FR-18 도구별 독립 색상·굵기 기억 (v1)
- 펜·하이라이터·도형·텍스트 각각의 마지막 색상·굵기를 독립 저장(Epic Pen Pro "differentiated stroke size" 대응). [근거 2]

### FR-19 고스트 모드 (v1)
- 툴바·UI를 모두 숨기고 전역 단축키만으로 도구를 사용하는 모드. [근거 2, 7]
- 발표/녹화 시 화면을 깔끔하게 비우는 용도.

### FR-20 페이딩 잉크 (v2)
- 그린 스트로크가 일정 시간 후 서서히 사라짐(자동 페이드아웃). [근거 2]
- 타이머/페이드 길이 설정값 제공. *(원본 정확 수치 §10-4 미확인 → 기본 3초, 1~10초 범위 자체 정의)*

### FR-21 하이라이트 커서 (v2)
- 커서 주위에 컬러 헤일로(강조 원) 표시 — 발표 시 마우스 위치 강조. [근거 4]

### FR-22 넘버링 태그 도구 (v2)
- 클릭마다 1,2,3… 번호가 매겨진 원형 태그를 찍는 도구(절차 설명용). [근거 12]

### FR-23 도형 채움 상태 순환 (v2)
- 같은 도형 도구 반복 클릭으로 채움 상태 순환: 없음 → 컬러 채움 → 외곽선 → 흰색 채움 → 검정 채움(ppInk 패턴). [근거 12]

### FR-24 돋보기 / 줌 (Optional, v2)
- 화면 일부를 확대해 보여주는 돋보기 창. *(Epic Pen 원본 제공 여부 미확인 — §10-1. 자체 기획 기능으로 분류)*

## 7. 기술 아키텍처

### 7.1 권장: WPF / .NET (네이티브) — ★1순위
근거: Epic Pen에서 영감받은 검증된 오픈소스 gInk(MIT, 100% C#/.NET WinForms)·포크 ppInk가 동일 접근으로 클릭관통·멀티디스플레이·휴대성을 달성. Electron 대비 입력 지연·메모리·패키징 측면 유리. [근거 12, 13]

**투명 오버레이 생성(검증된 표준):** [근거 8, 9, 10]
```xml
<Window x:Class="ScreenPen.OverlayWindow"
        WindowStyle="None"          <!-- 비클라이언트 영역 제거: 필수 -->
        AllowsTransparency="True"    <!-- 픽셀 단위 알파 투명 -->
        Background="Transparent"     <!-- 빈 영역 완전 투명 -->
        Topmost="True"
        ShowInTaskbar="False"
        ResizeMode="NoResize">
    <Canvas x:Name="InkCanvas"/>     <!-- InkCanvas 또는 커스텀 드로잉 -->
</Window>
```
- WPF는 내부적으로 레이어드 윈도우(`WS_EX_LAYERED`)의 **Application-Provided Content 모드(UpdateLayeredWindow)** 만 사용하므로 픽셀 단위 알파 + 안티에일리어싱이 자동 확보됨(컬러키 방식 미사용). [근거 9]
- 레이어드 윈도우는 최상위 윈도우에 적용 → **모니터당 오버레이 윈도우 1개** 생성(FR-17). [근거 8]

**클릭관통 메커니즘(핵심 원리):** [근거 11]
- OS의 Raw Input Thread(RIT)는 레이어드 윈도우에서 **커서 아래 픽셀이 100% 투명이면 그 윈도우를 건너뛴다** → 빈 영역은 자동으로 아래 앱에 입력 전달.
- 즉 "그린 영역=불투명=입력 캡처 / 빈 영역=투명=자동 통과"가 기본 동작.
- 단, **이미 그려진 잉크(불투명 픽셀) 위로는 입력이 캡처**되므로, 통과 모드(FR-09)에서는 윈도우 전체에 `WS_EX_TRANSPARENT`를 강제 설정해 모든 입력을 통과시킨다(고스트 모드와 연동).

**무설치 패키징:**
```powershell
dotnet publish -c Release -r win-x64 --self-contained true `
  -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true
```
- 단일 self-contained exe → .NET 런타임 미설치 PC에서도 실행(NFR-01/07).
- 설정은 `Path.Combine(AppContext.BaseDirectory, "settings.json")`에 저장(FR-13).

### 7.2 대안: Electron
- 투명 프레임리스 윈도우(`transparent:true, frame:false`) + 항상 위.
- 클릭관통은 WPF의 자동 픽셀 통과와 달리 **수동 구현**: 마우스 위치를 감시해 커서 아래 인터랙티브 요소 유무에 따라 `setIgnoreMouseEvents()`를 토글(electron-transparency-mouse-fix 패턴). [근거 14]
- 단점: 픽셀 단위 자동 관통이 없어 마우스 추적 로직 필요, 번들 용량 큼(NFR-07 불리), 입력 지연 가능.
- ⚠️ 기각된 주장: "setIgnoreMouseEvents가 항상 윈도우 전체에만 전역 적용된다", "forward 옵션은 Windows 전용이다", "electron-transparency-mouse-fix가 드래그앤드롭을 지원한다" → **기술 근거로 사용 금지**. [근거 14, 기각]

### 7.3 참조 오픈소스 (벤치마킹 권장)
| 프로젝트 | 라이선스 | 스택 | 참고점 |

## 8. 핵심 기술 리스크 및 대응

### 8.1 ★클릭관통 딜레마 (최우선 리스크)
- **문제**: "항상-위 오버레이로 그리기"와 "동시에 아래 앱 클릭"을 **하나의 상태로 동시 달성 불가**. 그린 잉크 위에서는 입력이 오버레이에 캡처되기 때문. [근거 11, caveat-6]
- **대응**: FR-09의 명시적 **모드 토글**(그리기 ↔ 통과)로 분리. 통과 모드에서 `WS_EX_TRANSPARENT` 강제. Epic Pen·ppInk도 동일하게 고스트 모드/핫키 토글로 해결. → 본 제품의 정식 설계로 채택.

### 8.2 멀티모니터·혼합 DPI
- 모니터별 배율 상이 시 좌표/굵기 왜곡 위험 → Per-Monitor V2 DPI Awareness 적용, 좌표를 물리 픽셀 기준으로 통일.

### 8.3 전체화면·보안 데스크탑
- 독점 전체화면(게임)·UAC 프롬프트·잠금화면 위에는 일반 오버레이가 표시 불가할 수 있음(OS 제약). → "전체화면 위 동작"은 일반 창모드/테두리없는 전체화면 한정으로 명시(§10-3 미확정).

### 8.4 전역 핫키 충돌
- 다른 앱이 동일 핫키 점유 시 등록 실패 가능 → 등록 실패 감지·사용자 재매핑 UI 제공.
